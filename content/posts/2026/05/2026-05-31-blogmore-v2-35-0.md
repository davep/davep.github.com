Well, I wasn't expecting to release a new version of
[BlogMore](https://blogmore.davep.dev/) quite so soon, given I made a
release earlier today. But [an issue came
in](https://github.com/davep/blogmore/issues/572) that the Bluesky icon
wasn't showing in the socials section on someone's blog, despite the fact
that the docs suggested it should work.

Sure enough, [the icon wasn't there](https://blog.cpjobling.net/) despite
the docs giving Bluesky as an example that should work. A quick check, and
the reason for the problem was clear: [the code was using a version of
FontAwesome that didn't have that
icon](https://github.com/davep/blogmore/pull/573/changes). An easy fix.

It's interesting to note that Copilot/Claude (which is what did this work
originally) produced one thing in the code, and then made an unsupportable
claim in the documentation. Furthermore, it *didn't* construct the tests to
at least match the documentation's claims. None of this is a surprise, of
course, but it is a good illustration of the fact that agent-based coding is
just fancy auto-complete with no real ability to maintain a coherent view of
the whole project.

Anyway, [v2.35.0](https://blogmore.davep.dev/changelog/#v2350) is now
available and I too can have a [Bluesky
link](https://bsky.app/profile/davep.org) in my socials. Oddly, I hadn't
thought to add it before now.
