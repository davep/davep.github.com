I've bumped [Gemtext](https://gemtext.davep.dev/) to
[v1.0.0](https://gemtext.davep.dev/changelog/#v100). I've gone from "this is
test code" to "this is stable code" mostly because the library is so simple
and I'm unlikely to add any radically new features to it[^1].

This bump also has one small addition: I've added [an `alt_text`
property](https://gemtext.davep.dev/gemtext/#gemtext.PreFormatted.alt_text)
to [the `PreFormatted`
class](https://gemtext.davep.dev/gemtext/#gemtext.PreFormatted). This exists
to capture and make available any text that comes after the pre-formatted
text marker. For example, this text:

````gemtext
```python
print("Hello, World!")
```
````

will now result in:

```python
PreFormatted(content='print("Hello, World!")', alt_text='python')
```

If/when I [add syntax highlighting of pre-formatted text blocks to
Rogallo](https://github.com/davep/rogallo/issues/151), this new property
will make that possible.

[^1]: I was thinking about adding a Gemtext builder class at some point,
    but I can't see a use for it any time soon. Perhaps v2.0.0.
