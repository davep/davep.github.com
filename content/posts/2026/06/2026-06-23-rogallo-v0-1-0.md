I've just made public, and published to [PyPI](/tag/pypi/), v0.1.0 of
[Rogallo](https://github.com/davep/rogallo). As mentioned a couple of times
recently, it's a terminal-based client for the [Gemini
Protocol](https://geminiprotocol.net/).

![Rogallo displaying the Gemini home page](/attachments/2026/06/23/rogallo.webp#centre)

This is a *very* early release, [with lots of things still to be
added](https://github.com/davep/rogallo/issues?q=sort%3Aupdated-desc%20is%3Aissue%20is%3Aopen%20label%3ATODO).
Right now it's at a place where it interacts with the basics of the
protocol, handles [the document
format](https://geminiprotocol.net/docs/gemtext-specification.gmi), and has
the basic navigation facilities you'd expect from a document browser. Things
I aim to add over the next few weeks include, in no specific order:

- A bookmark facility
- Support for the protocol's user input facility (status `1x`)
- Support for the protocol's client authentication facility (status `6x`)
- Support for local browsing (I want to be able to view and navigate local
  `gmi` files)
- A settable home page
- The ability to clear down the location history
- A cache system

I'll also be updating the TODO list as other ideas come to mind.

Another thing fairly high on the list is documentation. For now, Rogallo
should be pretty easy to grasp, and there is a help screen that describes
all the keyboard bindings in the current context.

![The help screen](/attachments/2026/06/23/rogallo-help.webp#centre)

Commands that can be run can also be discovered via the command palette.

![The command palette](/attachments/2026/06/23/rogallo-commands.webp#centre)

For those who care about different looks, there is, of course, support for
loading up different themes.

![The theme options](/attachments/2026/06/23/rogallo-themes.webp#centre)

This means that even folk who, for some reason, like light themes, can go
light too.

![Rose pine theme](/attachments/2026/06/23/rogallo-rose-pine.webp#centre)
![Latte theme](/attachments/2026/06/23/rogallo-latte.webp#centre)

Not really my thing, but I'm told some people like this.

If any of this sounds interesting and you want to have a play, Rogallo is
licensed GPL-3.0 and available [via
GitHub](https://github.com/davep/rogallo) and also [via
PyPI](https://pypi.org/project/rogallo/). If you have an environment that
has `pipx` installed, you should be able to get up and running with:

```sh
pipx install rogallo
```

It can also be installed using
[`uv`](https://docs.astral.sh/uv/getting-started/installation/):

```sh
uv tool install rogallo
```

If you don't have `uv` installed, you can use [uvx.sh](https://uvx.sh) to
perform the installation. For GNU/Linux, macOS, or similar:

```sh
curl -LsSf uvx.sh/rogallo/install.sh | sh
```

or on Windows:

```sh
powershell -ExecutionPolicy ByPass -c "irm https://uvx.sh/rogallo/install.ps1 | iex"
```

Once installed, run the `rogallo` command.

As I say above, this is *very* early days for the project, so don't be
surprised if you find something missing, or even run into a bug or two. If
you need help, have any ideas, or find any problems, please feel free to
[raise an issue](https://github.com/davep/rogallo/issues) or [start a
discussion](https://github.com/davep/rogallo/discussions).
