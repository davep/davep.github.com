Given I've been on a little bit of an [Emacs Lisp](/tag/emacs-lisp/) side
quest, it's been a couple or so days since I made a release of
[BlogMore](https://blogmore.davep.dev/). Today's release comes after [a
feature request about draft
posts](https://github.com/davep/blogmore/issues/355).

While support for marking posts as drafts, and including or excluding them
from a build, is something that's been in BlogMore from the start, it's not
something I've ever used. These days, when I'm writing a post, especially if
it's one that's taking a while to write, I'll do it in a branch and
eventually PR into `main` before publishing. Given this it was useful to get
a request relating to the feature as it helps me understand how someone else
might use it.

So as of [v2.7.0](https://blogmore.davep.dev/changelog/#v270), if a post is
[marked as a draft](https://blogmore.davep.dev/writing_a_post/#draft), and
[if drafts are included in the
build](https://blogmore.davep.dev/configuration/#include_drafts), it will be
pretty obvious:

![A draft post](/attachments/2026/04/01/draft-post.png#centre)

When the post's title appears in the archive it will also appear obvious
that it's still a draft too:

![A draft post in the archive](/attachments/2026/04/01/draft-archive.png#centre)

All of this is, of course, modifiable via the [template
API](https://blogmore.davep.dev/template-api/#draft-post-visual-indicator)
and via styling, so if the choice of colour or icon doesn't suit it can be
modified to taste.
